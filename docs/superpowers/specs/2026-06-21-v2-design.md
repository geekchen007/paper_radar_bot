# Paper Radar Bot v2 设计文档

## 目标

在现有单查询单格式基础上，增加三项能力：
1. 将 README 与设计文档改写为中文
2. 通过 `topics.yaml` 支持多领域、多关键词配置（OR 合并查询）
3. 默认输出 HTML 报告，可选切换为 Markdown

## 范围

### 本次包含

- `topics.yaml` 领域配置文件，支持每个领域多个关键词
- 多领域按顺序抓取，结果合并进同一份报告并按领域分节
- HTML 渲染器：单文件、内嵌 CSS、无外部依赖、GitHub 风格
- `OUTPUT_FORMAT` 环境变量控制输出格式（`html` 默认，`markdown` 可选）
- `topics.yaml` 不存在时回退到 `.env` 中的 `ARXIV_QUERY`（向后兼容）
- README.md 改写为中文
- 设计文档改写为中文

### 本次不包含

- Web UI 或可视化面板
- 跨领域去重
- 深色模式 CSS
- Jinja2 或其他模板引擎
- PDF 导出

## 数据结构

### Topic 数据类

```python
@dataclass
class Topic:
    """单个领域的配置。"""

    name: str              # 领域显示名，如"LLM Translation"
    keywords: list[str]    # 关键词列表，OR 合并查询
    max_results: int = 10  # 该领域最大抓取数量

    @property
    def arxiv_query(self) -> str:
        """将关键词列表拼接为 arXiv 查询串。"""
        return " OR ".join(f'all:"{kw}"' for kw in self.keywords)
```

### topics.yaml 格式

```yaml
topics:
  - name: LLM Translation
    keywords:
      - LLM translation
      - large language model translation
      - neural machine translation
    max_results: 10

  - name: Omni Model
    keywords:
      - omni model
      - multimodal model
      - vision language model
      - unified model
    max_results: 10

  - name: Sign Language Translation
    keywords:
      - sign language translation
      - sign language recognition
      - gesture recognition
      - sign language production
    max_results: 10
```

以上为默认内容（当 `topics.yaml` 不存在时自动生成）。

## 配置

### .env / 环境变量

| 变量 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `OPENAI_API_KEY` | 是 | — | OpenAI 兼容 API 密钥 |
| `OPENAI_BASE_URL` | 否 | `https://api.openai.com/v1` | API 基础 URL |
| `OPENAI_MODEL` | 否 | `gpt-4o-mini` | 模型名称 |
| `ARXIV_MAX_RESULTS` | 否 | `10` | 全局回退最大结果数 |
| `ARXIV_QUERY` | 否 | `cat:cs.AI` | 仅在 topics.yaml 不存在时使用 |
| `OUTPUT_FORMAT` | 否 | `html` | 输出格式：`html` 或 `markdown` |
| `REPORT_TIMEZONE` | 否 | `Asia/Shanghai` | 时区标签（信息性） |
| `REPORT_LOCALE` | 否 | `zh_CN` | 地区标签（信息性） |

### topics.yaml（独立文件）

位于项目根目录，管理领域列表。`.env` 只存放密钥和格式偏好，不再包含 `ARXIV_QUERY` 领域配置（但保留作回退）。

## 数据流

```
1. 加载 Config（.env）
2. 加载 Topics（topics.yaml；不存在则用 ARXIV_QUERY 构造单条默认 Topic）
3. 对每个 Topic：
   a. fetch_papers(topic.arxiv_query, topic.max_results) → list[Paper]
   b. 对每篇 Paper: summarize(paper, config) → Summary（失败标记，不中断）
4. render_report(topics_with_results, format) → str
   - format == "html"     → html_renderer.render_report()
   - format == "markdown" → renderer.render_report()
5. write_report(content, date_str, format)
   - html     → reports/2026-06-21.html
   - markdown → reports/2026-06-21.md
```

## HTML 渲染规格

### 页面结构

```html
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>Paper Radar 日报 — YYYY-MM-DD</title>
  <style>/* 内嵌 CSS */</style>
</head>
<body>
  <header>
    <h1>Paper Radar 日报 — YYYY-MM-DD</h1>
    <div class="meta">运行日期 · 领域数 · 论文总数</div>
    <nav>LLM Translation | Omni Model | Sign Language Translation</nav>
  </header>

  <section id="llm-translation">
    <h2>LLM Translation <span class="count">（N 篇）</span></h2>
    <article class="paper">
      <h3>论文标题</h3>
      <p class="paper-meta">作者 · 发布时间 · <a href="...">arXiv</a></p>
      <p class="abstract">中文摘要</p>
      <div class="highlights">
        <strong>关键亮点：</strong>
        <ul>...</ul>
      </div>
      <div class="applications">
        <strong>潜在应用方向：</strong>
        <ul>...</ul>
      </div>
    </article>
    <!-- 更多论文 -->
  </section>

  <!-- 更多领域 section -->
</body>
</html>
```

### CSS 规格

- 最大宽度 `860px`，水平居中
- 字体 `system-ui, -apple-system, sans-serif`
- 正文字号 `16px`，行高 `1.6`
- 每篇论文：`1px solid #e1e4e8` 边框，`4px` 圆角，`12px` 内边距，`16px` 底部外边距
- 领域标题（`h2`）：`#0366d6` 蓝色，下边框分隔线
- 论文标题（`h3`）：`#24292e` 深色
- `.paper-meta`：`#6a737d` 灰色，`14px`
- 总结失败提示：`background: #f6f8fa`，`color: #6a737d`，斜体
- 顶部导航：横排链接，`|` 分隔

## 错误处理

与 v1 保持一致，新增：

- `topics.yaml` 解析失败（YAML 格式错误）：退出并输出清晰提示
- `topics.yaml` 中某个 Topic 的 `keywords` 为空列表：跳过该 Topic 并输出警告
- `OUTPUT_FORMAT` 值非法（非 `html`/`markdown`）：退出并输出可用选项
