# AI 算法团队新人培训手册

欢迎加入团队。本手册是你入职后第一个月最重要的参考资料。

每个章节的结构是统一的：**规范说明 → ❌ 反例（附问题分析）→ ✅ 正例（附要点注释）→ ✏️ 操作 Checklist**。

建议按顺序阅读，读完一章就在实际工作中应用，不要一口气读完。

## 入职第一个月的学习节奏

| 时间 | 重点 |
|------|------|
| 第 1 周 | 读第一章（代码规范），提交第一个 PR |
| 第 2 周 | 读第二章（文献调研），完成一篇论文的结构化笔记 |
| 第 3 周 | 读第三章（技术汇报），准备第一次组会分享 |
| 第 4 周 | 读第四、五章（汇报 + 周报），建立固定汇报节奏 |

## 使用原则

- **规范不是束缚，是降低沟通成本的共识**。当你不确定"这样写对不对"，先查本手册。
- **反例比正例更重要**。看懂为什么不好，才能真正避免。
- **Checklist 要真的用**。每次提交代码、发周报之前，过一遍对应的 Checklist。
- **有疑问随时问导师**，但先在手册里查一遍。
