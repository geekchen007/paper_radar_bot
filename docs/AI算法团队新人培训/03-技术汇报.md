# 第三章：技术汇报规范

> 汇报不是"展示你做了多少工作"，而是"让听众做出更好的决策"。

---

## 1. 汇报前的准备

### 规则

**开始做 PPT 之前，先回答三个问题：**

1. **听众是谁？** 技术同学（可以讲细节）还是管理层（只要结论和影响）？
2. **他们最关心什么？** 方法是否可行？进度是否正常？需要什么支持？
3. **汇报结束后我希望发生什么？** 获得认可？得到资源？推动决策？

**准备"电梯演讲"（30秒版本）：** 在开始做 PPT 之前，先能口头说清楚：

> "我在做 [什么]，解决了 [什么问题]，目前 [进展和结果]，接下来 [计划]。"

如果这30秒说不清楚，PPT 也做不好。

---

### ❌ 反例

```
常见的错误准备方式：

1. 直接打开 PPT，把代码截图、实验日志粘贴进去
2. 花了80%时间在背景和相关工作上，只剩5分钟讲自己的工作
3. PPT 做了50页（对着写论文），每页都密密麻麻
4. 没有彩排，不知道自己要讲多久
5. 对着技术 Leader 汇报，却在解释"什么是 Transformer"

结果：听众听了30分钟还不知道你做了什么，你想表达什么。
```

---

### ✅ 正例

```
正确的准备流程：

第一步：确定听众和目的（5分钟）
→ 听众：技术 Leader + 2名同组同学（都了解基础知识）
→ 目的：汇报消融实验结果，获得是否继续这条路线的决策

第二步：写出电梯演讲（10分钟）
→ "我在做工业缺陷检测的小目标提升，解决了 Baseline YOLOv8
   在 5mm 以下缺陷漏检率高的问题。目前在测试集上 Recall
   从 61% 提升到 74%，实验证明 FPN+注意力机制是有效的，
   接下来计划在真实产线数据上验证。"

第三步：规划 PPT 结构（10分钟）
→ 用纸列出每页要讲什么，确认15分钟内讲完

第四步：做 PPT（1-2小时）

第五步：彩排（15分钟）
→ 对着计时器讲一遍，检查时间和逻辑是否顺畅
```

---

### ✏️ Checklist

- [ ] 是否明确了听众背景（技术 vs 管理层）
- [ ] 是否能用30秒口头说清楚汇报内容
- [ ] PPT 总页数是否与汇报时间匹配（大约1-2分钟/页）
- [ ] 是否做过至少一次计时彩排

## 2. Slide 结构规范

### 规则

**标准技术汇报结构（以15分钟为例）：**

| 页数 | 内容 | 时间 |
|------|------|------|
| 1 | 封面（标题 + 汇报人 + 日期） | 30秒 |
| 2 | **结论先行**：核心结果是什么（不要让听众等到最后） | 1分钟 |
| 3-4 | 背景与问题定义（为什么要做这件事） | 2分钟 |
| 5-7 | 方法（我做了什么） | 4分钟 |
| 8-10 | 实验结果（效果怎么样） | 4分钟 |
| 11 | 结论与下一步 | 2分钟 |
| 12 | 备用页（Q&A 时可能用到的细节） | — |

**每页 Slide 的黄金规则：**

- **标题即结论**，不写泛泛的类别词
- **一页只讲一个观点**
- **正文不超过 5 行**，每行不超过 15 个字
- **字体不小于 24pt**（后排的人能看到）

---

### ❌ 反例

```
❌ 反例标题（无信息量）：
- "实验结果"
- "方法介绍"
- "数据集"
- "下一步工作"

❌ 反例 Slide 内容：
标题：实验结果
正文：
  我们在 COCO 数据集上进行了大量实验，采用了多种数据增强方式，
  包括随机翻转、随机裁剪、颜色抖动等，同时对比了多个 Baseline，
  在不同的评价指标上进行了比较，实验结果表明我们提出的方法在大
  多数情况下优于 Baseline，证明了方法的有效性，后续将继续优化...
  （密密麻麻一段话，听众边听边读，两件事都没做好）
```

---

### ✅ 正例

```
✅ 正例标题（结论型）：
- "FPN + 注意力机制使小目标 Recall 从 61% 提升至 74%"
- "数据增强策略对比：Mosaic 在小目标场景下效果最佳"
- "在真实产线数据上 Precision 下降 8%，主要原因是光照变化"
- "下一步：3个方向的实验计划，预计2周内出结果"
```

**一页规范 Slide 的样式（文字描述）：**

```
┌─────────────────────────────────────────────────────┐
│  标题（大字，结论型）：                               │
│  "FPN Neck 是小目标提升的关键：去掉后 AP 下降 5.3%"  │
│                                                     │
│  左侧：方法对比示意图（简洁，只保留关键组件）          │
│                                                     │
│  右侧：消融实验表格                                  │
│  ┌──────────────┬────────┬──────────┐               │
│  │ 配置          │  AP   │  小目标AP │               │
│  ├──────────────┼────────┼──────────┤               │
│  │ Baseline      │ 48.2  │   21.3   │               │
│  │ + FPN         │ 52.7  │   26.8   │  ← 重点标红   │
│  │ + FPN + Attn  │ 53.9  │   28.1   │               │
│  └──────────────┴────────┴──────────┘               │
│                                                     │
│  底部一句话总结：FPN 贡献最大（+4.5 AP），            │
│  注意力机制进一步提升 1.3 AP                         │
└─────────────────────────────────────────────────────┘
```

---

### ✏️ Checklist

- [ ] 第2页是否放了"结论先行"（核心结果）
- [ ] 每页标题是否是结论型（不是类别词）
- [ ] 每页正文是否不超过 5 行
- [ ] 字体是否不小于 24pt
- [ ] 是否删除了所有与核心论点无关的内容

## 3. 图表规范

### 规则

**图表必须自说明——遮住正文，只看图表也能理解。** 判断标准：一个不了解你工作的同事，只看这张图能不能读懂？

**不同信息用不同图表类型：**

| 信息类型 | 推荐图表 |
|---------|---------|
| 方法对比（离散） | 表格 |
| 趋势变化（训练曲线） | 折线图 |
| 分布差异 | 柱状图 |
| 系统结构 | 架构示意图 |
| 定性结果展示 | 图像对比（左Baseline，右我们的方法） |

---

### ❌ 反例

```python
# 生成了一张图，但完全无法自说明
import matplotlib.pyplot as plt
plt.plot(losses)
plt.savefig("loss.png")
```

生成的图存在以下问题：
- 纵轴无标签（是 loss？是 accuracy？）
- 横轴无标签（是 epoch？是 step？）
- 没有图例（有多条线时无法区分）
- 没有标题
- 纵轴范围不合理（从 0.9998 到 1.0001，看不出差异）

---

### ✅ 正例

```python
import matplotlib.pyplot as plt
import matplotlib.ticker as ticker

fig, ax = plt.subplots(figsize=(8, 5))

ax.plot(train_losses, label="Train Loss", color="#2196F3", linewidth=2)
ax.plot(val_losses,   label="Val Loss",   color="#F44336", linewidth=2, linestyle="--")

ax.set_xlabel("Epoch", fontsize=13)
ax.set_ylabel("Loss", fontsize=13)
ax.set_title("Training and Validation Loss (FPN + Attention, lr=1e-3)", fontsize=14)
ax.legend(fontsize=12)
ax.grid(True, alpha=0.3)

# 标注最低 val loss 点
best_epoch = val_losses.index(min(val_losses))
ax.annotate(
    f"Best: {min(val_losses):.4f} @ Epoch {best_epoch}",
    xy=(best_epoch, min(val_losses)),
    xytext=(best_epoch + 5, min(val_losses) + 0.02),
    arrowprops=dict(arrowstyle="->"),
    fontsize=11,
)

plt.tight_layout()
plt.savefig("outputs/loss_curve.png", dpi=150)
```

**表格规范示例（Markdown）：**

```markdown

（输入分辨率 800×1333，GPU：A100，FPS 为单卡推理速度）

| 方法             | Backbone    | AP   | AP₅₀ | AP₇₅ | APₛ  | APₘ  | APₗ  | FPS |
|-----------------|-------------|------|------|------|------|------|------|-----|
| Faster RCNN     | R-50-FPN    | 40.2 | 61.0 | 43.8 | 24.2 | 43.5 | 52.0 | 5   |
| YOLOv8-m        | CSPDarkNet  | 50.2 | 67.2 | 54.6 | 32.1 | 54.4 | 64.4 | 80  |
| **Ours (DFPN)** | R-50-FPN    | **53.9** | **70.1** | **58.2** | **28.1** | **57.3** | **66.8** | 35 |

```

### ✏️ Checklist

- [ ] 所有图表是否有标题
- [ ] 折线图/柱状图是否有轴标签和单位
- [ ] 多条线/多组数据是否有图例
- [ ] 表格是否有表头说明和脚注
- [ ] 重点数据是否用颜色/加粗突出
- [ ] 图像对比是否左右排列（Baseline 在左，我们的方法在右）

## 4. 口头表达规范

### 规则

**开场30秒决定听众的注意力能持续多久。** 先说结论，再说过程。

**标准开场话术模板：**

> "今天我要汇报的是 **[主题]**。
> 核心结论是：**[一句话结论 + 关键数字]**。
> 接下来我会花约 **[X]** 分钟介绍背景、方法、实验，然后讲下一步计划。"

**提问环节的应对原则：**

- 知道答案 → 直接回答，用数据支撑
- 不确定 → "这个问题很好，我去查一下再回复您"（绝不瞎猜）
- 问题超出范围 → "这不在我这次研究的范围内，我后续可以调研一下"

---

### ❌ 反例

```
错误开场（花了4分钟介绍背景，听众已经开始看手机）：

"目标检测是计算机视觉中的一个重要任务，它的目标是在图像中
定位并识别所有感兴趣的目标。随着深度学习的发展，从 R-CNN、
Fast R-CNN、Faster R-CNN，到后来的单阶段方法 SSD、YOLO，
目标检测的精度和速度都有了很大提升。近年来，Transformer 的
引入进一步推动了这一领域的发展……（持续3分钟）
然后，我们的方法是……"

错误应对提问：
"这个嘛……应该是……我觉得大概是因为……可能是这样的……"
（一直在猜，说了很多但没有数据支撑）
```

---

### ✅ 正例

```
正确开场（30秒内说完）：

"今天我汇报缺陷检测小目标提升的实验进展。
核心结论是：在我们的测试集上，Recall 从 Baseline 的 61%
提升到了 74%，提升了 13 个百分点，同时推理速度保持在 35 FPS。
接下来我用15分钟介绍具体的方法和实验，最后说下一步计划。"

正确应对提问：
问："你的方法在光照变化下鲁棒性如何？"
答："这个问题很好。从我们目前的数据来看，在我们的标准测试集上
    没有评估光照变化，但在真实产线数据的初步实验里，不同光照
    条件下精度有8%的波动。我下周会专门做一组光照鲁棒性实验，
    到时候给大家一个正式的数据。"
```

---

### ✏️ Checklist

- [ ] 开场是否在30秒内说出了核心结论
- [ ] 是否没有逐字念 Slide
- [ ] 汇报时间是否在规定时间的 ±2 分钟以内
- [ ] 对不确定的问题是否回答"去确认后回复"而非瞎猜
- [ ] 汇报结束后是否发送了 Slide 给参会者

## 5. 常见错误清单（Top 10）

以下是应届生第一次汇报最高频的失误，对照检查：

| # | 错误 | 正确做法 |
|---|------|---------|
| 1 | 逐字念 Slide | Slide 是提示，用自己的话讲，多看听众 |
| 2 | 背景讲太长，自己的工作只有5分钟 | 背景不超过总时间的20%，重点在方法和结果 |
| 3 | 结果页只展示数字，不说结论 | 每张结果图都要说"这说明了什么" |
| 4 | 严重超时（没有彩排） | 上台前必须计时彩排至少一次 |
| 5 | 字体过小，后排看不清 | 正文不小于 24pt，图表标签不小于 16pt |
| 6 | 动画和特效过多 | 除非有教学目的，不用动画 |
| 7 | 图表缺少轴标签、图例、单位 | 图表必须能自说明 |
| 8 | 只报喜不报忧（隐藏问题） | 主动说明失败的实验和遇到的问题，才能获得帮助 |
| 9 | "下一步计划"太模糊（"继续优化"） | 写具体的任务和时间点（"下周三前完成X实验"） |
| 10 | 提问时说"我不知道"就结束了 | 说"我不知道，但我会去查，X天内回复您" |
